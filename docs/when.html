Project Justification

For the workload of this project, it will be divided by the strong suits that each individual member believes they accel at, as highlighted in the section further down. We will then assess the value of each task that is placed on each member, to weigh which ones are more important than other tasks or vice versa. This is to ensure that the tasks divided are fair and correlate with the strength of each respective member. Through this, we can continue with what is expected with each members workload:

  • Each person is expected to have spent roughly one hour each day of this project, resulting in an estimated 7 hours of work dedicated to this project a week
  • Each person attends all meetings each week and relays information to other group members and the respective tutor to confirm that things are going according to plan, and if not, used the time to help with issues expressed
  • Each person stays on top of their work on a weekly basis so that things can continue smoothly and not interrupt any other members workflow for other classes.
  • Each person clearly understands what tasks they are allocated with, resulting in less confusion and unnecessary work such as writing a part twice.

While this would be considered the optimal way to go about this, sometimes there will be obstacles that might need to be addressed (actual workloads):

  • Dates of certain assignments or work gets pushed back resulting in a readjustment of proximity of due dates.
  • Perceptions of difficulty that each member may have with their tasks which could result in dedicating longer amounts of time (or not)
  • Personal reasons that may conflict with their expected workload which could result in less time being dedicated (or not)
  • Other assignments from other classes that demand more attention or are urgent to finish, as each member has a set of different classes from others. This can result in not being 100% focused during each expected time period

Resources & Tools

Microsoft Teams

We will use MS Teams to hold face-to-face meetings between all members as well as tutor/lecturer throughout the various phases of the project. This platform will be beneficial for group members to clearly discuss and share ideas as opposed to communicating through email/text messaging.

GitHub

GitHub is a code hosting platform that will allow all members to contribute and make changes to our web based project. As our project will require HTML and JavaScript, all group members will be able to have access to all files/code related to creating our assignment. Other platforms that could be used for real-time code collaboration include Brackets or Visual Studio Code

Trello

Trello is a collaborative board where team members are able to organise, delegate and plan their work using a visual platform. Our group will aim to plan our work/phases using a collaborative Trello board. This will be done in order to keep track of tasks that need to be completed, delegate tasks to each member and show to members of the group what each individual will be or is working on. Ultimately, Trello will play a large part in monitoring our progress and ensuring we remain on track throughout the development process.
